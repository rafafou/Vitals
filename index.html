Vitals · Security — in detail

It scans every server handler (OnServerEvent / OnServerInvoke), scores your security /100, and can insert the missing check in one click.

#SeverityWhat it catches
R1CRITICALA server handler placed inside a LocalScript — it never runs server-side.
R2CRITICALAn unvalidated client argument used on currency, stats, position or DataStores.
R5HIGHThe server acts on an Instance supplied by the client without checking it.
R3MEDIUMA client argument used with no type or range validation.
R4MEDIUMNo rate-limit / debounce — the remote can be spammed.
R6LOWA RemoteFunction with no clear return — can hang the caller.
R7INFOA remote with no server handler — forgotten attack surface.
R8INFOHandler wired via a named function — flagged for manual review.
R9INFOAn UnreliableRemoteEvent used where reliable state is expected.
🩺

Vitals · Memory — in detail

Two engines in two tabs: a static pass on your connection hygiene, and a live memory monitor during playtest.

🔌 Static — Connections

Flags risky patterns without running the game:

L1 a :Connect with discarded return created inside a loop or a high-frequency handler (Heartbeat, Touched…).

L4 Instance.new in a repeating scope with no :Destroy().

L6 many discarded connections and no Maid/Trove/Janitor cleanup system.

📈 Live — Memory

During a playtest (F5), it samples memory every second and draws a sparkline per category:

Total · Instances · Signals · Lua Heap · Script

Each gets a verdict: stable ✅, or “rising ⚠” when it only goes up — the signature of a leak. Rising Signals = piling connections; rising Instances = objects never destroyed.

Vitals · Performance — in detail

It scans the whole place, scores its optimization /100, and tells you exactly what to lighten. Click an alert to select the objects.

#SeverityWhat it finds
O1MEDIUMModuleScripts never required — dead code.
O2LOWScripts with identical code (copy-paste).
O3LOWEmpty or disabled scripts — dead weight.
O4MEDIUMUnionOperations (CSG) — heavy and slow to load.
O5INFOGhost parts: transparent + no collision + no children.
O6INFOMany unanchored parts — needless physics cost.
O7MEDIUM1000+ parts and StreamingEnabled off — slow load (biggest win).
O8INFO200+ decals / textures / images — graphics memory.
O9LOWMeshParts with Precise collision — costly to compute.
